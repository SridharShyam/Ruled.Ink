# Contributing to Ruled.Ink

First off, thank you for considering contributing to Ruled.Ink! It's people like you that make Ruled.Ink such a great tool for students and productivity enthusiasts.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct (be kind, be professional).

## How Can I Contribute?

### Reporting Bugs

- Use the **Bug Report** template.
- Describe the steps to reproduce the bug.
- Include details about your browser and OS.

### Suggesting Enhancements

- Use the **Feature Request** template.
- Explain why this enhancement would be useful.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the project builds successfully (`npm run build`).
4. Make sure your code follows the existing style (we use Tailwind CSS v4 and React 19).
5. Issue that pull request!

## Local Development

```bash
# Clone the repository
git clone https://github.com/SridharShyam/Ruled.Ink.git

# Install dependencies
npm install

# Start the dev server
npm run dev
```

## Design Principles

- **Notebook Aesthetic**: Keep it warm, cream-colored, and serif-heavy.
- **Privacy First**: All data stays in `localStorage`. No external databases.
- **Focused UX**: Minimize distractions. Use subtle animations.

Thank you!
